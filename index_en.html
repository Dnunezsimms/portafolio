Engineering Practices

Methodological and development standards applied to all my projects.

Clean Folder Architecture

Projects are structured following professional software engineering guidelines to separate immutable business logic from execution scripts and configurations:

  • src/: Packaged and immutable Python modules (e.g., ingestion, processing, and calculation logic).
  • config/: Dynamic parameter files to avoid hardcoded variables.
  • scripts/: Sequentially orchestrated driver flows (e.g., 01_inventory.py, 02_build_sites.py).
  • tests/: Suite of logical validations to verify code functionality after modifications.

Decoupled Parameters

In the WineWise-RS project, all vineyard assignments, meteorological study variables, and local/remote directories are centralized in YAML configuration files. This allows:

  • Porting the pipeline to other servers simply by modifying config/paths.yaml.
  • Adjusting study variables (e.g., maximum/minimum temperatures or radiation) in config/study_config.yaml without rewriting the validation logic.
  • Preventing security gaps by separating API keys (stored in environment variables or unversioned local .env files).

Logging & Production Traceability

To ensure that data ingestions from APIs like DataVid are auditable, formal logging is implemented in operational scripts:

  • Detailed log of HTTP calls, retries on connection failures, and weather server timeouts.
  • Tracking inconsistencies in consolidated hourly logs for manual audit.
  • Persistent logs structured in files (e.g., dashboard_obj3_integrado.log) with timestamps and severity levels (INFO, WARNING, ERROR).

Unit Testing for Critical Algorithms

The accuracy of thermal accumulation and station mapping depends on mathematical algorithms that require logical consistency. Pytest is used to ensure exactness:

  • Grid Validation: Tests to verify that ERA5-Land lat/lon coordinates are correctly associated with the nearest vineyard pixel.
  • Formula Integrity: Unit tests on Growing Degree Days (GDD) biofix calculation and winter chill portions.
  • Prevents software regressions when optimizing the mathematical calculation engine.

Raw Data Immutability (QA/QC)

A fundamental principle of the workflow is respect for the scientific truth of field observations:

  • The data/raw/ directory is treated as a strict read-only database. No script can alter it.
  • The entire gap-filling process and filtering of faulty sensors generate new intermediate files in data/processed/.
  • The gap-filling logic is documented through explicit flowcharts (such as gap_filling_decision_flow.png) to maintain absolute transparency on climate data manipulation.
